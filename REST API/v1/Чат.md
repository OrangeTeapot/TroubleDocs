> За флагом `NEXT_PUBLIC_CHAT_ENABLED`. При выключенном — все роуты `404`. DM строго между
> друзьями (`canAccessChat`; unfriend рвёт доступ ретроактивно); нарушение доступа → `404`.

### GET /api/v1/chats
Мои диалоги. **Ответ (`result`) — `ChatSummaryDTO[]`:**
```ts
{ chatId; kind:"dm"|"team"; title; slug:string|null; avatarUrl; hue;
  lastMessage:{ body:string|null; createdAt; mine }|null; unreadCount:number }
```
(сорт по последнему сообщению; DM видны только пока стороны — друзья).

### POST /api/v1/chats
Открыть/получить чат. **Запрос (`openChatSchema`):** `{ kind:"dm"|"team", profileId?, teamId? }`.
**Ответ:** `{ chatId }` (201 создан / 200 уже есть). **Ошибки:** 404 таргет не найден / не друг /
не участник команды / таргет запретил DM (`messageVisibility=NONE`); 422 self-DM; 429 (≤10 новых DM/мин).

### GET /api/v1/chats/[id]/messages
**Запрос (query):** `limit` (1–50, деф. 30), `cursor` (назад, по createdAt desc),
`after` (вперёд для поллинга, по createdAt asc).
**Ответ (`result`):** `{ items: MessageDTO[]; nextCursor:string|null }`,
`MessageDTO { id; authorId; authorName; authorAvatarUrl; authorHue; body:string|null;
deleted:boolean; createdAt; mine }`. **Заметки:** при первой странице/поллинге обновляет
`lastReadAt`; удалённые — `body:null, deleted:true`.

### POST /api/v1/chats/[id]/messages
**Запрос (`sendMessageSchema`):** `{ body }` (1–2000 после санации: срез C0-control кроме `\t`/`\n`
+ DEL). **Ответ:** `MessageDTO` (201). **Ошибки:** 422 пусто/длинно; 429 (≤5 сообщений/10с).

### DELETE /api/v1/chats/[id]/messages/[messageId]
Soft-удаление СВОЕГО сообщения (`deletedAt`, тело стирается). **Ответ:** `{ deleted: true }`.
**Ошибки:** 404 не автор / не в этом чате (не палим).
