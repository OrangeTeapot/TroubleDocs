### Конверт ответа
Все роуты `/api/v1/**` (кроме явно отмеченных) возвращают единый конверт
(`lib/api/respond.ts`):
```jsonc
{
  "result": <полезная нагрузка> | null,   // null при ошибке
  "meta": {
    "isSuccess": true,                      // false при ошибке
    "errors":   [{ "code": "string", "description": "string" }],
    "warnings": [{ "code": "string", "description": "string" }],
    "info":     [{ "code": "string", "description": "string" }]
  }
}
```
Ниже в каждом роуте указывается только форма `result` при успехе. При ошибке
`result = null`, `meta.isSuccess = false`, детали в `meta.errors`.

### Аутентификация
Большинство `/api/v1/**` требуют активную сессию (cookie Better Auth). Без сессии —
`401 { code: "unauthorized" }`. Сессия → пользователь → профиль; если у пользователя нет
профиля, часть роутов вернёт `404 { code: "no_profile" }`.

### Коды статусов
| Код | Значение |
|-----|----------|
| 200 | Успех |
| 201 | Создан ресурс (POST create/upload) |
| 400 | Битый JSON / неверный параметр |
| 401 | Нет сессии |
| 403 | Нет прав (напр. не владелец) |
| 404 | Не найдено / нет доступа (намеренно не палим существование) |
| 409 | Конфликт состояния (дубль, уже участник, нет заявки) |
| 413 | Слишком большой файл |
| 415 | Неверный тип файла |
| 422 | Ошибка валидации (Zod) или сущность не найдена по ссылке в теле |
| 429 | Rate-limit |
| 502/503 | Сбой/неконфигурированность внешнего сервиса (Steam/Xbox) |

### Разрешение `[id]`
Идеи и команды в URL принимают **слаг ИЛИ cuid** (`findIdeaBySlugOrId`,
`teamSlugOrIdWhere`) — старые id-ссылки продолжают работать.

### Фиче-флаги
Часть поверхностей за `NEXT_PUBLIC_*`-флагами (инлайнятся на сборке): чат
(`NEXT_PUBLIC_CHAT_ENABLED`), Discord-вход (`NEXT_PUBLIC_DISCORD_ENABLED`). При выключенном
флаге соответствующие роуты отвечают `404`.
