DTO идеи (`IdeaDTO`):
```ts
{ id; slug; title; description; minPlayers; maxPlayers; scheduledAt; durationMinutes;
  status; visibility:"PRIVATE"|"TEAM"|"PUBLIC"; peopleScope:"NONE"|"FRIENDS"|"FRIENDS_OF_FRIENDS"|"LIST"|"ANYONE";
  withoutGames; suggestGame; pinned; avatarUrl; requiredSources:GameSource[]; crossPlayOnly;
  games:{ id; name; iconUrl }[];
  invites:{ profileId; profileSlug; displayName; username; avatarUrl }[];
  inviteProfileIds: string[];
  author:{ id; slug; displayName; avatarUrl }; createdAt }
```

### GET /api/v1/ideas
- **Auth:** сессия. **Запрос (query):** `q`, `visibility`, `suggestGame` (bool),
  `gameId`, `limit` (деф. 24, макс 100), `cursor`.
- **Ответ (`result`):** `{ items: IdeaDTO[]; nextCursor: string|null }`.
- **Заметки:** видны только доступные зрителю идеи (PUBLIC; свои; TEAM где приглашён).
  Курсорная пагинация (createdAt desc, id desc). Фильтры складываются (AND).

### POST /api/v1/ideas
- **Auth:** сессия. **Запрос (JSON, `createIdeaSchema`):** `title` (3–100), `description`
  (≤1000, nullable), `minPlayers`/`maxPlayers` (1–100), `scheduledAt` (ISO, nullable),
  `durationMinutes` (15–1440, деф. 120), `visibility` (деф. PUBLIC), `peopleScope` (деф. ANYONE),
  `withoutGames`, `suggestGame`, `gameIds` (≤50), `inviteProfileIds` (≤50), `requiredSources`, `crossPlayOnly`.
- **Ответ (`result`):** `IdeaDTO` (201). **Ошибки:** 422 `game_not_found`/`profile_not_found`/
  maxPlayers<minPlayers. **Заметки:** title авто-генерится из первой игры, если не задан;
  слаг уникализируется; нужен хотя бы один из title / gameIds / withoutGames / suggestGame.

### GET /api/v1/ideas/[id]
Одна идея (slug или id). **Ответ:** `IdeaDTO`. **Ошибки:** 404 если нет доступа
(`canViewIdea`: PUBLIC всем; PRIVATE только автору; TEAM автору+приглашённым).

### PATCH /api/v1/ideas/[id]
Только автор. **Запрос (JSON, `updateIdeaSchema`):** поля как в create + `pinned`.
**Ответ:** `IdeaDTO`. **Ошибки:** 403 не автор; 422 валидация. **Заметки:** переданные
массивы (gameIds/inviteProfileIds/…) ПОЛНОСТЬЮ замещают прежние (в транзакции).

### DELETE /api/v1/ideas/[id]
Только автор. **Ответ:** `{ id: string }`.

### POST /api/v1/ideas/[id]/avatar · DELETE /api/v1/ideas/[id]/avatar
Обложка идеи, только автор. POST: `multipart` file → `{ avatarUrl }` (201).
DELETE → `{ avatarUrl: null }`. Ошибки: 403 не автор.

### GET /api/v1/ideas/[id]/matches
Прямой подбор игроков. **Ответ (`result`) — `PlayerMatch[]`:**
```ts
{ profileId; profileSlug; displayName; avatarUrl; languages:string[]; percent:number;
  breakdown:{…}; presence:string; busy:string }
```
- **Ошибки:** 400 `not_applicable` (visibility≠PUBLIC или peopleScope=NONE); 404 нет доступа.
- **Заметки:** охват по `peopleScope`; учитывает `matchVisibility` кандидата; `languages`
  отдаются только если позволяет приватность; cap 500 кандидатов.

### GET /api/v1/ideas/[id]/game-suggestions
Обратный подбор игр под ростер. **Ответ (`result`):**
```ts
{ participants:{ id; displayName }[];
  suggestions:{ score; isFree; ownedCount; wantLicenseCount; missingCount; wontPlayCount;
                participantCount; games:{ id; name; iconUrl; crossPlay }[] }[] }
```
- **Ошибки:** 400 `not_applicable` (suggestGame≠true); 404 нет доступа.
- **Заметки:** ростер по visibility (PRIVATE=[автор]; TEAM/приглашённые=[автор+invites];
  PUBLIC без invites=публичные профили); группировка по сигнатуре, топ-20.

### GET /api/v1/ideas/[id]/suggest
Единый эндпоинт: игроки + игры + лучшая группа. **Ответ (`result`):**
```ts
{ players: PlayerMatch[];            // пусто если visibility≠PUBLIC или peopleScope=NONE
  games: GameSuggestionOut[];        // пусто если suggestGame=false
  participants: { id; displayName }[];
  group: { size; score; belowMin; players:PlayerMatch[]; games:GameSuggestionOut[] } | null;
  peopleScope: string }
```
- **Заметки:** `group=null`, если нет кандидатов с `percent>0`; лучшая группа = топ-N (N≤maxPlayers)
  с percent>0 + игры под состав (автор + выбранные).
