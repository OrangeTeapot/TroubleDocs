## Аутентификация (Better Auth)

### ANY /api/auth/[...all]
Catch-all, делегирован библиотеке **Better Auth** (`toNextJsHandler`). Конфиг —
`lib/auth.ts` (email+password, плагин username, Discord-провайдер при заданных
`DISCORD_CLIENT_ID/SECRET`). Не использует конверт `{result,meta}` — формат Better Auth.

Основные эндпоинты:
- `POST /api/auth/sign-up/email` — регистрация email+пароль (в закрытой бете заблокирована
  при `BETA_DISABLE_SIGNUP=true`; публичная регистрация идёт через `/api/v1/beta/signup`).
- `POST /api/auth/sign-in/email` — вход по email+паролю.
- `POST /api/auth/sign-in/username` — вход по нику+паролю (плагин username).
- `POST /api/auth/sign-out` — выход.
- `GET  /api/auth/get-session` — текущая сессия.
- `POST /api/auth/request-password-reset` / `POST /api/auth/reset-password` — сброс пароля по токену.
- `POST /api/auth/change-email` — смена email с подтверждением на новый адрес.
- `POST /api/auth/update-user` — смена `name` / `username` / `displayUsername`.
- `POST /api/auth/sign-in/social` (`provider: "discord"`) + callback — Discord OAuth
  (только при включённом провайдере; новые аккаунты в закрытой бете блокируются хуком).
- `POST /api/auth/link-account` / `unlink-account` — привязка/отвязка соц-провайдера.

Параметры пароля: длина 8–72, проверка состава. Rate-limit Better Auth: 100 запросов / 60с.

## Регистрация в бете и OAuth-привязки

### POST /api/v1/beta/signup
Self-service регистрация по инвайт-коду (публичная, без сессии). Активна только при
`BETA_INVITE_SIGNUP=true` (иначе `404`).
- **Запрос (JSON, `betaSignupSchema`):**
  - `email` — string, валидный email, ≤254 (обяз.)
  - `password` — string, 8–72, состав ≥3 классов (`passwordSchema`) (обяз.)
  - `name` — string 2–32 (обяз.)
  - `username` — string 3–30, `[a-z0-9_]`, reserved-слова запрещены (обяз.)
  - `inviteCode` — string 4–64 (обяз.)
  - `consent` — literal `true` (обяз.)
- **Ответ (`result`):** `{ created: true }` (+ Set-Cookie сессии при авто-входе).
- **Ошибки:** 403 `invite_invalid`/`beta_full`; 409 `dup` (email/username занят); 422 валидация; 400 битый JSON.
- **Заметки:** код потребляется атомарно (conditional UPDATE); жёсткий потолок аккаунтов
  (`betaMaxAccounts()`); пароль хэшируется argon2id вне транзакции; User+Account+Profile
  создаются в транзакции; авто-вход best-effort.

### GET /api/v1/auth/steam/start
- **Auth:** не требуется. **Запрос:** нет.
- **Ответ:** HTTP-redirect на Steam OpenID. realm/return — `publicOrigin(req)` (учитывает
  прокси/`BETTER_AUTH_URL`). Callback → `/api/v1/auth/steam/callback`.

### GET /api/v1/auth/steam/callback
- **Auth:** нет (callback Steam). **Запрос:** query OpenID-ассерции.
- **Ответ:** redirect:
  - залогинен → привязка Steam к профилю → `/settings?steam=linked` (или `?steam=taken`).
  - Steam уже привязан к юзеру → вход → `/profile`.
  - новый Steam + регистрация открыта + не достигнут потолок → создаёт синтетический
    аккаунт (`steam_<id>@steam.local`), импортирует библиотеку (при `STEAM_API_KEY`), вход → `/profile`.
  - ошибки → `/login?steam=error|closed|full`.
- **Заметки:** сессионная cookie подписывается HMAC-SHA256 (формат Better Auth).

### GET /api/v1/auth/xbox/start
- **Auth:** **требуется сессия** (Xbox — только привязка к залогиненному). **Запрос:** нет.
- **Ответ:** redirect на OAuth OpenXBL (xbl.io); ставит CSRF-cookie `xbox_oauth_state`
  (httpOnly, 600с). `404`, если `XBOX_OAUTH_PUBLIC_KEY` не задан.

### GET /api/v1/auth/xbox/callback
- **Auth:** проверяет сессию + CSRF-state. **Запрос:** query `state` + пер-юзер токен
  (`app_key`/`token`/`apiKey`/`code`).
- **Ответ:** redirect `/settings?xbox=linked|taken|error` (или `/login` без сессии).
- **Заметки:** пер-юзер токен используется разово (аккаунт + импорт титулов), НЕ хранится;
  проставляет `xboxXuid/xboxGamertag/xboxVerified=true`, импортирует игры Xbox.
